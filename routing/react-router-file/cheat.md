The "Single File" ConventionLike TanStack, React Router v7 uses a convention where a single route file handles the UI, the data, and the actions.The Config: Use the routes.ts file to define your tree (or use the optional file-based routing plugin).The Difference: React Router uses a "Bifurcated" build—some of your code in the file runs only on the server, and some runs only on the client.2. Topic 6: Auth (The loader Redirect)In React Router, you don't have beforeLoad. Instead, you use the loader itself to protect the route.The Pattern: Inside your loader, check the session. If it's missing, throw redirect("/login").The Benefit: Since the loader runs on the server (by default), the unauthorized user never even downloads the data or the component for the protected page.3. Topic 3: Search Params (The Web Standard)React Router leans heavily on the browser's native URLSearchParams object.Validation: Like Next.js, it doesn't have a built-in validateSearch hook like TanStack. You must manually parse the URL in the loader.The Experience: You'll use new URL(request.url).searchParams to get your data.4. Topic 4: Data Loading (The loader Function)This is the closest competitor to TanStack's loader.The Loader: Export a loader function from your route file.The useLoaderData Hook: Access that data in your component.HTML Forms: React Router excels at "Actions." If you want to delete a post, you don't use axios.post; you use a native <Form method="post">.5. Topic 5: Pending & Error StatesError Boundaries: Export an ErrorBoundary function from the same file. React Router will automatically catch errors from the loader or the component and render this instead.Hydration: Since React Router v7 defaults to Server-Side Rendering (SSR), your "Pending" state is often handled by the browser's loading bar unless you use Optimistic UI or useNavigation().Comparison: The Final Stand-offFeatureTanStack RouterReact Router (v7)Primary EnvironmentClient (SPA)Server (SSR) / ClientSearch Param ValidationBuilt-in (Zod/Adapter)Manual (inside loader)Type Safety100% Automatic (generated)Strong, but requires ComponentProps or clientLoader typesData FlowLoader -> ComponentServer Loader -> ComponentForm HandlingManual (useMutation)Native <Form> and action
